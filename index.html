Proyecto Personal

Este proyecto surge como una iniciativa personal para mejorar y replantear una idea técnica que desarrollé tiempo atrás. En su momento, fue parte de un ejercicio que no tuvo el resultado que esperaba, por lo que decidí retomarlo y enfocarme en perfeccionar tanto el diseño de la base de datos como la experiencia de usuario.

La aplicación consiste en una única vista que permite registrar asistencias docentes. Incluye un encabezado con botones para desplegar un menú lateral y acceder a opciones de gestión de usuario. El formulario principal utiliza los tipos de inputs adecuados y realiza validaciones en tiempo real, tanto al deseleccionar un campo como al momento de enviar los datos, mostrando alertas en caso de errores.

Este proyecto fue desarrollado con HTML, estilos personalizados en CSS, el framework Bootstrap y JavaScript para manejar la interacción del usuario y validaciones del formulario.

Demokratica

Demokratica es una plataforma de votación en línea orientada a facilitar la toma de decisiones colaborativas en grupos de personas, permitiendo llegar a un consenso. El sistema permite la creación de sesiones personalizadas, el uso de distintos métodos de votación y la gestión de participantes de forma segura y transparente.

Mis responsabilidades técnicas principales:

  • Integración de la funcionalidad principal de votación:
    • Lógica de conexión frontend-backend para renderizado dinámico de métodos de votación.
    • Envío y recepción de peticiones HTTP a través de endpoints REST.
    • Aplicación de la lógica de negocio para garantizar el correcto funcionamiento del sistema de votación.
  • Integración inicial de la pasarela de pagos MercadoPago:
    • Creación y configuración de cuenta de desarrollador.
    • Conexión del frontend con el backend para la selección de productos y procesamiento de pagos.
    • Gestión del flujo de compra según los planes ofrecidos en la plataforma.
  • Implementación de trazabilidad y auditoría de operaciones sensibles en la base de datos mediante Hibernate Envers:
    • Seguimiento de cambios críticos en votaciones, usuarios y transacciones.
    • Registro automático de auditoría para análisis posteriores.
  • Implementación de vistas del frontend:
    • Página de “Conócenos”.
    • Página de "Planes" (modelo de cobro por sesión e invitados).
  • Diseño UI/UX:
    • Centro de usuario (dashboard personal).
    • Interfaz de gestión de cuenta.
  • Refactorización de código para mejorar la seguridad:
    • Eliminación de variables sensibles quemadas en el código.
    • Uso adecuado de variables de entorno tanto en frontend como backend.

Tecnologías utilizadas:

  • Frontend: React, Next.js (TypeScript), TailwindCSS, ESLint.
  • Backend: Spring Boot (Java), Hibernate (incluyendo Envers), REST API.
  • Base de datos: PostgreSQL.

Herramientas y servicios de desarrollo:

  • Gestión de tareas: Jira, Notion.
  • Control de versiones: Git (con GitHub).
  • Prototipado y diseño: Penpot.
  • Despliegue y hosting: Vercel (frontend), Render (backend), Neon (base de datos PostgreSQL en la nube).

ZenUN

ZenUN es una aplicación web desarrollada como parte de un proyecto académico, orientada a facilitar la gestión de préstamos de elementos y la organización de eventos comunitarios ofrecidos por la Dirección de Bienestar de la Facultad de Ingeniería de la Universidad Nacional de Colombia.

Mis responsabilidades técnicas principales:

  • Implementación de los menús principales de navegación y un menú desplegable accesible desde cualquier sección de la aplicación para mejorar la experiencia de usuario.
  • Apoyo en el diseño y modelado de la base de datos relacional utilizando MySQL, asegurando consistencia e integridad en las entidades y relaciones clave.
  • Gestión de la trazabilidad y auditoría de operaciones realizadas sobre la base de datos, implementando mecanismos para el registro y análisis de actividades relevantes.
  • Diseño y ejecución de casos de prueba funcionales para evaluar flujos de interacción clave y validar respuestas esperadas del sistema bajo distintos escenarios de uso.
  • Uso de herramientas para la generación de informes dinámicos que permitieran analizar tendencias de uso y comportamiento en los préstamos y participación en eventos.

Tecnologías utilizadas:

  • Frontend: HTML, CSS, JavaScript.
  • Backend: Django.
  • Base de datos: MySQL.

Herramientas y servicios de desarrollo:

  • Gestión de tareas: Jira.
  • Control de versiones: Git (con GitHub).
  • Prototipado y diseño: Figma.
  • Despliegue y hosting: Render (front y back), CleverCloud (base de datos).
